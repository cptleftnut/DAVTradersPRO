## 2024-05-24 - Expensive Formatting in WebSocket Streams
**Learning:** In React components that subscribe to high-frequency WebSocket streams (like Binance `@ticker`), using `.toLocaleString()` or instantiating `new Intl.NumberFormat()` inside the message handler creates unnecessary garbage collection pressure and CPU spikes, as these functions are expensive to initialize repeatedly.
**Action:** Always pre-instantiate and cache `Intl.NumberFormat` formatters outside the component body or use `useMemo` for static configurations when they are invoked within hot paths (like loops or frequent stream handlers).
