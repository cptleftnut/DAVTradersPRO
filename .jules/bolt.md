## 2024-05-18 - Caching Intl.NumberFormat in High-Frequency Streams
**Learning:** Re-creating `Intl.NumberFormat` objects or calling `Number.prototype.toLocaleString()` inside high-frequency operations (like WebSocket `onmessage` handlers processing tick streams, or `framer-motion` `useTransform` calls calculating animation frames) causes excessive CPU overhead and layout recalculations in React, ultimately blocking the main thread.
**Action:** When formatting numbers dynamically in fast-updating UI components, define pre-configured `new Intl.NumberFormat(locale, options)` objects outside the component logic/render loops, and reuse their `.format(value)` method.
